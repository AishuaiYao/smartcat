# SmartCat - AI图像采集小程序

基于微信小程序的智能图像采集与管理平台，支持设备连接、实时视频流采集、编码器数据记录与云端存储。

## 系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        微信小程序 SmartCat                        │
├─────────────────────────────────────────────────────────────────┤
│  首页(home) │ 设备页(device) │ 数据采集(collect) │ 相册(album)  │
├─────────────────────────────────────────────────────────────────┤
│                     微信云开发 (数据库/存储)                      │
├─────────────────────────────────────────────────────────────────┤
│                    TCP Socket (命令/控制)                         │
├─────────────────────────────────────────────────────────────────┤
│                    UDP Socket (视频流)                           │
├─────────────────────────────────────────────────────────────────┤
│                     ESP32 智能机器人                             │
│            (摄像头 │ 电机驱动 │ 编码器 │ WiFi)                   │
└─────────────────────────────────────────────────────────────────┘
```

## 功能特性

### 基础功能

| 功能 | 说明 |
|------|------|
| 设备管理 | 自动搜索、添加、管理ESP32设备 |
| 实时图像采集 | 通过UDP协议实时获取摄像头视频流 |
| 单帧拍照 | 通过TCP协议获取单帧高清图像 |
| 数据集管理 | 本地存储(最多500张)与云端同步 |
| 云端存储 | 基于微信云开发的数据集管理 |
| 批量操作 | 支持批量上传、删除、下载 |
| 远程控制 | 实时遥控机器人移动采集数据 |

### 进阶功能

| 功能 | 说明 |
|------|------|
| 编码器采集 | 实时记录左右电机编码器增量值(各1000个) |
| 环形缓冲区 | 高效存储编码器数据，自动覆盖旧数据 |
| 柱状图生成 | Canvas绘制双电机编码器对比图 |
| 相册保存 | 将柱状图直接保存到手机相册 |

## 页面说明

### 首页 (home)

- 设备列表展示已添加的ESP32设备
- 点击搜索按钮探测局域网内的设备
- 设备卡片显示名称、图标、在线状态
- 支持设备添加与删除

### 设备页 (device)

- TCP连接管理，握手协议
- 单帧图像采集(320x120灰度)
- 实时日志显示连接状态
- 跳转数据采集和相册页面
- 编码器图表保存功能

### 数据采集页 (collect)

- UDP视频流实时接收与显示
- 分片协议重组完整帧
- 电机控制：前进、左转、右转、停止
- 速度调节：0-100%
- **编码器数据采集**：记录左右电机脉冲增量
- **柱状图保存**：退出时自动保存到相册

### 相册页 (album)

- 本地缓存图片管理(最多500张)
- 云端图片同步与管理
- 批量上传、下载、删除
- 图片预览与标注

## 通信协议

### TCP协议 (命令控制)

```
连接端口: 5000
握手命令: HELLO\n
拍照命令: CAPTURE
开始采集: START:<speed>
停止采集: STOP
左转命令: MOTOR_LEFT
右转命令: MOTOR_RIGHT
直行命令: MOTOR_FORWARD
速度调节: SPEED:<0-100>
开始视频: STREAM
停止视频: STOP_STREAM
UDP端口通知: UDP_HELLO:<port>
```

### UDP协议 (视频流)

```
本地端口: 5002
目标端口: 5001
帧格式: [帧号高8位][帧号低8位][总分片数][分片索引][图像数据...]
首分片额外数据: [电压][电机A][电机B][编码器A高][编码器A低][编码器B高][编码器B低]
帧大小: 160x120 = 19200 字节
```

### 设备响应协议

```
握手响应: <设备名>|<图标>|<MAC地址>
UDP就绪: UDP_OK
UDP未就绪: UDP_NOT_READY
```

## 编码器数据流

```
┌──────────────┐     ┌─────────────────┐     ┌──────────────┐
│  ESP32编码器   │ ──▶ │  首分片解析      │ ──▶ │  环形缓冲区   │
│  A/B脉冲计数   │     │  提取脉冲增量    │     │  1000个增量值 │
└──────────────┘     └─────────────────┘     └──────────────┘
                                                      │
                                                      ▼
┌──────────────┐     ┌─────────────────┐     ┌──────────────┐
│  手机相册     │ ◀── │  Canvas柱状图    │ ◀── │  退出时保存   │
│  保存图片     │     │  双电机对比      │     │  图表数据     │
└──────────────┘     └─────────────────┘     └──────────────┘
```

### 溢出处理

- ESP32编码器每32768清零
- 增量计算时自动跳过大跳变(阈值±16000)

## 快速开始

### 1. 环境准备

- 微信开发者工具
- 开通微信云开发
- ESP32设备(固件: `E:\AICleanBot\main\app_main.cpp`)

### 2. 项目配置

1. 导入项目到微信开发者工具
2. AppID: `wxe43671e5bc6f25c1`
3. 在开发者工具中开通云开发
4. 创建数据库集合 `dataset`
5. 设置集合权限为"所有用户可读写"

### 3. 运行项目

1. 编译并预览
2. 连接ESP32 WiFi热点(默认IP: 192.168.4.1)
3. 首页点击➕搜索设备

## 使用说明

### 设备连接

1. 连接ESP32 WiFi热点
2. 首页点击搜索按钮
3. 等待设备响应，点击确认添加
4. 点击设备卡片进入控制面板

### 实时视频采集

1. 点击"数据采集"进入实时画面
2. 使用方向按钮控制机器人移动
3. 调节速度滑块(0-100%)
4. 点击开始/停止自动采集图像

### 编码器数据记录

1. 进入数据采集页面，自动开始记录
2. 控制机器人移动采集编码器数据
3. 最多记录1000个编码器增量值
4. 退出页面时自动保存柱状图到相册

### 查看编码器图表

1. 进入设备详情页
2. 点击"保存图表"按钮
3. 图表自动保存到手机相册

### 云端同步

1. 切换到可联网的网络(WiFi/移动数据)
2. 点击"相册" > 切换到"云端"Tab
3. 选择图片进行上传/下载/删除操作

## 数据格式

### 本地存储

```javascript
// 已保存图片
wx.getStorageSync('savedImages')
// 编码器图表数据
app.globalData.encoderChartData = { dataA, dataB, count }

// 图片结构
{
  path: String,      // 文件路径
  time: String,      // 保存时间
  uploaded: Boolean  // 是否已上传云端
}
```

### 云端数据库 (dataset集合)

```javascript
{
  _id: String,
  imageId: String,   // 图片唯一标识
  label: String,     // 分类标签
  uploaded: Boolean, // 上传状态
  createTime: Number // 创建时间戳
}
```

## 注意事项

- 数据采集时需连接设备WiFi
- 上传云端需切换到可访问互联网的网络
- 本地缓存最多500张图片(约10MB)
- 编码器数据在退出采集页时自动保存
- 云端数据库需配置正确的读写权限
- UDP视频流依赖微信小程序createUDPSocket API
